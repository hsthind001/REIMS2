STEP 1: VERIFY PREREQUISITES (2 minutes)
---
$ docker --version        # Should show: Docker version 24.0+
$ docker-compose version  # Should show: Docker Compose v2+
$ git --version          # Should show: git version 2.40+
$ python --version       # Should show: Python 3.13+
$ node --version         # Should show: v20+

$ cd /path/to/REIMS2
$ docker-compose ps      # All services should be "Up"

If not installed:
- Docker: https://docs.docker.com/get-docker/
- Python: https://python.org/downloads/
- Node: https://nodejs.org/

STEP 2: INSTALL TASKMASTER (1 minute)
---
$ pip install taskmaster-ai

Verify:
$ taskmaster --version
# Should show: TaskMaster v1.x.x

STEP 3: CONFIGURE API KEY (1 minute)
---
You need EITHER Claude OR OpenAI API key.

For Claude (recommended):
$ export CLAUDE_API_KEY="your-key-here"

For OpenAI (alternative):
$ export OPENAI_API_KEY="your-key-here"

Make permanent:
$ echo 'export CLAUDE_API_KEY="your-key"' >> ~/.bashrc
$ source ~/.bashrc

Get API key:
- Claude: https://console.anthropic.com/
- OpenAI: https://platform.openai.com/api-keys

STEP 4: DOWNLOAD PRD FILES (1 minute)
---
All PRD files should be in your Downloads folder already.

Organize them:
$ cd /path/to/REIMS2

$ mv ~/Downloads/REIMS_MASTER_PRD.txt taskmaster_prd/
$ mv ~/Downloads/SPRINT_01_FOUNDATION_PRD.txt taskmaster_prd/

STEP 5: INITIALIZE TASKMASTER (3 minutes)
---
$ cd /path/to/REIMS2
$ taskmaster init

This creates .taskmaster/ directory.

Load configuration:
$ taskmaster config load taskmaster_prd/TASKMASTER_CONFIG.yaml

Scan project:
$ taskmaster context scan
# Takes 2-3 minutes, analyzes your codebase

================================================================================
TWO EXECUTION MODES
================================================================================

AUTOMATED MODE (Faster, Less Control):
---
$ taskmaster start

TaskMaster will:
- Execute all tasks automatically
- Make decisions without human approval
- Commit changes automatically
- Continue until sprint complete

Recommended for: Sprints 3-8 after you're comfortable with TaskMaster

INTERACTIVE MODE (Slower, More Control):
---
$ taskmaster start --interactive

TaskMaster will:
- Show proposed implementation for each task
- Wait for your approval before executing
- Show results after each task
- Wait for your validation before committing

Recommended for: Sprint 1-2 (first time using TaskMaster)

You can switch modes anytime:
$ taskmaster pause
$ taskmaster config set execution_mode interactive
$ taskmaster resume

================================================================================
MONITORING PROGRESS
================================================================================

REAL-TIME STATUS:
---
$ taskmaster status

Shows:
- Current sprint progress (X/Y tasks complete)
- Current task being executed
- Estimated time remaining
- Recently completed tasks
- Any errors or warnings

VIEW LOGS:
---
$ taskmaster logs --tail 50

Shows last 50 log lines. Useful for debugging.

For specific task:
$ taskmaster logs S1-05

VIEW METRICS:
---
$ taskmaster metrics show

Shows:
- Tasks completed vs planned
- Time spent vs estimated
- Test pass rate
- Code coverage
- Success rate

GENERATE REPORT:
---
$ taskmaster report sprint-1

Creates detailed sprint report in reports/ directory.

================================================================================
IF SOMETHING GOES WRONG
================================================================================

TASK FAILS:
---
TaskMaster will automatically:
1. Try to fix the error (up to 3 attempts)
2. If can't fix: Rollback changes
3. Pause and request human intervention
4. Provide context and suggested actions

You then:
1. Review the issue: taskmaster logs S1-XX
2. Fix manually if needed
3. Resume: taskmaster resume S1-XX

DOCKER ISSUES:
---
$ docker-compose ps      # Check service status
$ docker-compose logs backend  # Check for errors
$ docker-compose restart backend  # Restart service

DATABASE ISSUES:
---
$ docker-compose exec postgres pg_isready  # Check database
$ docker-compose exec backend alembic current  # Check migrations

NEED TO START OVER:
---
Rollback to clean state:
$ git checkout main
$ git branch -D sprint/01-foundation
$ taskmaster reset --confirm

Then start again from STEP 6.

TASKMASTER NOT WORKING:
---
$ taskmaster validate-env  # Check environment
$ taskmaster logs --debug  # Verbose logging
$ taskmaster help          # Show help

Still stuck? Review TASKMASTER_SETUP_GUIDE.txt for detailed troubleshooting.
